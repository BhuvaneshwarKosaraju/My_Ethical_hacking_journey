Lesson 1: Network Scanning with Nmap

What You’ll Learn:

  • Basics of Nmap (Network Mapper)
  • How to perform a network scan to discover devices on a network
  • Identify open ports and services running on a host
  • Understand service versions and OS detection

Why Nmap is Important:

Nmap is one of the most commonly used tools for network discovery and vulnerability scanning. Ethical hackers use Nmap to map out targets, identify open ports, and find services that might be vulnerable to attacks.

1.1: Introduction to Nmap

Nmap is an open-source tool used to discover devices and services running on a network. It's often used for network mapping, port scanning, and service enumeration.

1.2: Installing Nmap

  • Linux: Open your terminal and run: 
    sudo apt-get install nmap
  • Windows: Download and install from Nmap.org.
  • MacOS: Use Homebrew: 
    brew install nmap

1.3: Basic Nmap Scanning

The simplest Nmap scan can be performed by typing the following in your terminal:

nmap <target_IP_address>

Example:

nmap 192.168.1.1

1.4: Nmap Scan Types

SYN Scan (-sS)

Fast scan that doesn’t complete the TCP handshake (stealth scan).

nmap -sS 192.168.1.1

TCP Connect Scan (-sT)

Normal scan that completes the TCP handshake. It’s more detectable but works on systems that block SYN packets.

nmap -sT 192.168.1.1

Service Version Detection (-sV)

Identify services and versions running on the open ports.

nmap -sV 192.168.1.1

Operating System Detection (-O)

Guess the operating system of the target machine.

nmap -O 192.168.1.1

1.5: Example Exercise

Run a network scan on a local target (use your router or any machine within your local network):

nmap -sS -sV -O 192.168.1.1

This command will:

  • Scan the most common ports using a SYN scan
  • Detect versions of services running on open ports
  • Attempt to identify the operating system of the target

1.6: Advanced Nmap Scans

Scan a range of IPs:

nmap 192.168.1.1-100

Scan a subnet:

nmap 192.168.1.0/24

Scan a specific port:

nmap -p 22 192.168.1.1

Aggressive Scan: This scan performs multiple actions at once, including OS detection, service version detection, and more.

nmap -A 192.168.1.1

1.7: Conclusion

Nmap is a powerful and essential tool in any ethical hacker's toolkit. Mastering its use will allow you to map networks, discover services, and detect vulnerabilities that could be exploited. The more you practice, the better you’ll understand the target environments.

Hands-on Lab Exercise:

  1. Scan a Local Network:
    • Find out the IP range of your local network (use ipconfig on Windows or ifconfig on Linux/Mac).
    • Run an Nmap scan on the local network to identify devices.
    • Experiment with different scan options (e.g., SYN scan, service version scan).
  2. Remote Target Scan (with Permission):
    • Try scanning a publicly available target (with permission, such as a vulnerable VM or test environment).
    • Run various scans on the target to see which services are open.
